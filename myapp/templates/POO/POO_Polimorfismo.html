17.Polimorfismo

El polimorfismo es un principio fundamental de la Programación Orientada a Objetos (POO) que permite que objetos de diferentes clases respondan al mismo mensaje o método de manera diferente. En Python, el polimorfismo se logra mediante la implementación de métodos con el mismo nombre en diferentes clases, lo que permite que estos métodos sean invocados utilizando la misma sintaxis, independientemente del tipo de objeto al que se apliquen. Esto significa que un objeto puede mostrar diferentes comportamientos dependiendo de su tipo o clase.








17.1 Caracteristicas del Polimorfismo

El polimorfismo, como una de las características clave de la Programación Orientada a Objetos (POO), presenta varias características distintivas:

● Flexibilidad: Permite que un mismo método pueda actuar de manera diferente según el tipo de objeto al que se aplica, lo que brinda flexibilidad en el diseño de programas.

● Reutilización de código: Al escribir métodos genéricos que pueden manejar diferentes tipos de objetos, se promueve la reutilización del código, lo que ahorra tiempo y esfuerzo en el desarrollo de software.

● Simplicidad: Simplifica el diseño y la implementación de programas al permitir que el mismo método pueda ser invocado con diferentes tipos de objetos utilizando la misma interfaz.

● Abstracción: Permite que los objetos de diferentes clases compartan un mismo nombre de método, proporcionando una abstracción del comportamiento que es independiente de los detalles de implementación de cada clase.

● Interoperabilidad: Facilita la integración de diferentes componentes de software al proporcionar una interfaz común para interactuar con ellos, lo que mejora la interoperabilidad entre diferentes partes de una aplicación.

● Extensibilidad: Permite extender el comportamiento de una clase existente mediante la creación de nuevas subclases que sobrescriben los métodos heredados, lo que facilita la ampliación y la modificación del sistema sin modificar el código existente.







17.2 Ejemplo de Polimorfismo en Python

En este ejemplo, la clase Animal define un método sonido(), pero no proporciona una implementación concreta. Las clases Perro, Gato y Vaca heredan de Animal y sobrescriben el método sonido() con implementaciones específicas para cada tipo de animal. La función hacer_sonar() toma un objeto de tipo Animal y llama a su método sonido(). Cuando se llama a esta función con diferentes tipos de objetos (por ejemplo, Perro, Gato o Vaca), se ejecuta el método sonido() correspondiente a cada clase, demostrando así el polimorfismo en acción.
